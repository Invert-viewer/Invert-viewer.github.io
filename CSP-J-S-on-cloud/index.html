# “云” 上流程

# 报名

1. 报名时间

首先我应该参加 CSP-J 也就是普及组，无意间发现报名时间已经截止了……

算了，明年还可以再报名！

劝退指数 + 1

2. 报名方式

信息学竞赛，很明显是网络报名 ，但必须要通过教师认证，可我在的这个苏北小县城哪来有资格的教师😓，接着找办法报名，发现只可以 “各省认证负责人将帮助认证者完成报名并进行认证安排”，也就是说我还要单线去联系本省负责人……

这报名中的不确定因素有点多啊（我也好好感受了一下 “地域差异”）

算了，地域差异还是可以想办法克服的！

劝退指数 + 1

# 认证前

1. 内容了解

大概看了看普及组的认证内容，官网上面很浅显地提了一句：“涉及算法和编程”，第一眼看到时感觉不是太难嘛，毕竟《信息技术》选择性必修一里面还是有数据结构的内容的，然后当我具体去找那个 “编程 “代表什么的时候直接把我劝退，好家伙，居然是 C++！

😥其实还有 C 语言，但反正我都没学过

倍感无言之余，想起我第一个接触的编程语言可是 Python 啊，又很难沾上边……

算了，反正 C++ 还可以再学！

劝退指数 + 1

2.“到达” 考场

“第一轮认证以笔试为主”，当我看到这句话时就知道我肯定不能在自己家认证了。

😞好吧其实想想也不可能允许你开卷的……

接着便是找找认证地点安排，找到本市中的地点，发现认证地点距我家实在是太远了！

查高德地图发现即使是三个路线都超过 100 公里，就算驾车也要两个小时，但要真当天早上参加的话其实还好 (毕竟时间在下午 2 点)。

劝退指数 + 1

好吧其实这比起其他的不利因素这个还算友好。

# 感受

  • 到网络上去搜索关于信息学竞赛的相关事宜时，看到当下实行的政策，这真的给我泼了一盆冷水，“CSP-J/S 认证成绩优异者，可参加 NOI 省级选拔，省级选拔成绩优异者可参加 NOI。” 这告诉我好像有好几级台阶需要攀爬，而对于既没有教育资源又是竞赛强省的我来说属实…… 不太可能🙄
  • 算是彻底体验了一会 “地域差异” 为何物，感觉这好像就是天生给你设定好的，难道这是注定的吗？

没有人天生被定义为失败者！相信一定可以打破这困局！
